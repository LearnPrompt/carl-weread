# Workflow: digest-apply

## 触发

用户说：

- 我读完了，帮我消化
- 这是我的划线，帮我变成行动
- 把这章变成 Obsidian 笔记
- 从这次阅读里提炼一个选题

## 目标

把阅读从「看过」推进到「进入行动」。

## 输入

- 书名、章节名
- 用户划线/想法，或微信读书 `/book/bookmarklist` 返回的划线
- 最近项目/选题上下文

## 输出结构

```markdown
# 阅读行动卡｜《书名》章节名

## 一句话收获
...

## 它解释了我最近哪个问题
...

## 可以马上改的一个动作
...

## 可沉淀的原子笔记
[[概念名]]：...

## 可发展的内容选题
标题方向：...
切入角度：...
```

## 规则

1. 不写大而空的读书感想。
2. 每次只给一个最小行动，避免把阅读变成新的待办负担。
3. 原子笔记要能独立理解，不能只写「这很重要」。
4. 选题线索要服务内容生产，优先连接 AI 工具、Agent、工作流、实测案例。
5. 用户要求落盘时，写入真实 Obsidian vault，不写 mirror。
