# Output Style

## 核心表达

结论先行，少讲概念，多讲「为什么是这一节」和「读完做什么」。

## 推荐输出必须包含

1. 书名和章节名。
2. 为什么它连接用户最近的问题。
3. 一个读前问题。
4. 一个读后动作。
5. `weread://` 链接和可复制的 `open 'weread://...'` 命令。
6. 手动兜底路径：书名、章节名；必要时提醒安装并登录微信读书客户端。

## 避免

- 泛泛夸书。
- 一次推荐太多。
- 把阅读复盘写成年度报告。
- 「不是……而是……」这类高频 AI 句式。
- 只输出一个裸 `weread://` 文本，假设用户一定能直接打开。
